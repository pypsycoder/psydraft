# PsyDraft — Шаблон амбулаторной консультации психиатра

> Интерактивный офлайн-инструмент для подготовки психиатрических консультаций

## О проекте

**PsyDraft** — это веб-приложение для психиатров, которое помогает структурированно собирать клинические данные и автоматически формировать текст заключения. Инструмент работает полностью офлайн, запускается в любом современном браузере и не требует серверных зависимостей.

### Ключевые возможности

- **Сбор клинических данных** — структурированные формы для жалоб, анамнеза, психического статуса
- **Оценка психического статуса** — 10 подразделов с детализированными параметрами
- **Диагностика по МКБ-10** — поиск и выбор из 46+ психиатрических диагнозов
- **Библиотека рекомендаций** — 30+ готовых рекомендаций по категориям
- **Управление назначениями** — гибкие схемы приёма препаратов
- **Генерация заключения** — автоматическое формирование итогового текста

## Быстрый старт

### Требования

- Python 3.x (для сборки)
- Современный браузер (Chrome, Firefox, Safari, Edge)

### Сборка и запуск

```bash
# Клонировать репозиторий
git clone https://github.com/pypsycoder/psydraft.git
cd psydraft

# Собрать проект
python frontend/psychstatus/build.py

# Открыть в браузере
open frontend/psychstatus/dist/psychstatus.html
```

Скрипт `build.py` обрабатывает директивы `<!-- INCLUDE ... -->` и собирает модульные исходники в единый HTML-файл.

## Использование

1. **Откройте** `frontend/psychstatus/dist/psychstatus.html` в браузере
2. **Заполните** секции по порядку (жалобы → анамнез → статус → диагноз)
3. **Выберите** рекомендации из библиотеки или добавьте свои
4. **Добавьте** назначения с дозировками и схемами
5. **Скопируйте** готовое заключение в медицинскую документацию

### Горячие клавиши

- Навигация по секциям через боковое меню
- Сворачивание/разворачивание групп через `<details>`
- Редактирование итогового текста прямо в результате

## Разработка

### Добавление нового диагноза МКБ-10

Отредактируйте массив `ICD_CODES` в файле `src/js/ui/icd_search.js`:

```javascript
const ICD_CODES = [
    { code: "F20.0", name: "Параноидная шизофрения" },
    { code: "F32.1", name: "Депрессивный эпизод средней степени" },
    // Добавьте новый код здесь
];
```

### Добавление рекомендации

Добавьте в массив `RECOMMENDATIONS_DATA` в файле `src/js/ui/recommendations.js`:

```javascript
{
    id: "rec_new",
    category: "general",
    title: "Заголовок рекомендации",
    shortText: "Краткий текст",
    fullText: "Полный развёрнутый текст рекомендации"
}
```

### Изменение стилей

```css
    /* ... */
}
```

## Предложения по улучшению

### Приоритет: Высокий

#### 1. Сохранение и загрузка сессий
**Проблема:** Данные хранятся только в LocalStorage и теряются при очистке браузера.
**Решение:** Добавить экспорт/импорт JSON-файлов сессий для надёжного хранения и переноса между устройствами.

```javascript
// Пример API
function exportSession() {
    const data = collectAllFormData();
    const blob = new Blob([JSON.stringify(data)], {type: 'application/json'});
    downloadFile(blob, `session_${Date.now()}.json`);
}
```

#### 2. Печать заключения
**Проблема:** Нет оптимизированного режима печати.
**Решение:** Добавить CSS для `@media print` и кнопку "Печать" с форматированием для A4.

#### 3. Валидация обязательных полей
**Проблема:** Можно сгенерировать пустое заключение.
**Решение:** Подсветка незаполненных обязательных полей перед генерацией отчёта.

### Приоритет: Средний

#### 4. История изменений (Undo/Redo)
Добавить возможность отмены последних изменений для защиты от случайных ошибок.

#### 5. Шаблоны заключений
Готовые пресеты для типичных случаев:
- Первичная консультация
- Повторный приём
- Консультация для МСЭК
- Госпитализация

#### 6. Автосохранение с индикатором
Периодическое сохранение (каждые 30 сек) с визуальной индикацией статуса.

#### 7. Тёмная тема
Альтернативная цветовая схема для работы в условиях низкой освещённости.

#### 8. Расширение базы МКБ-10
Увеличить количество диагнозов с 46 до полного справочника раздела F (психические расстройства).

### Приоритет: Низкий (на перспективу)

#### 9. PWA (Progressive Web App)
Установка как приложение на рабочий стол с поддержкой офлайн через Service Worker.

#### 10. Статистика и аналитика
Локальная статистика по частоте диагнозов и назначений для анализа практики.

#### 11. Интеграция с МИС
API для интеграции с медицинскими информационными системами (опционально).

#### 12. Голосовой ввод
Использование Web Speech API для диктовки свободных полей.

#### 13. Многоязычность
Поддержка украинского и английского языков интерфейса.
