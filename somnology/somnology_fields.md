# Поля формы Somnology — техническая спецификация для переноса в МИС

Документ описывает все поля ввода модуля «Сомнологический прием» (PsyDraft Sleep) с их идентификаторами, типами и вариантами ответов — для маппинга данных при интеграции со сторонней медицинской информационной системой. Раздел «Советник» (диагностические подсказки) в документ намеренно не включен.

Источники: `somnology/src/sections/*.html`, `somnology/src/js/app.js`.

> Версия контракта: 2.0 (взрослый первичный приём). Поля ниже сохранены для обратной совместимости; новые структурированные поля перечислены в разделе 9. Советник не является источником диагноза и не изменяет план без явного действия врача.

Порядок разделов соответствует навигации `sections/sidebar.html` и порядку INCLUDE в `index.template.html` (раздел «Советник» между «Шкалы» и «Диагноз» пропущен по требованию задачи).

## 3. Шкалы (`#scales`, sections/scales.html)

Пункты шкал генерируются в JS (функция `createScaleItems()`) в контейнеры `#isi-items`, `#ess-items`, `#stopbang-items`; в HTML-разметке изначально пусты.

### 3.1. ISI — Insomnia Severity Index (Индекс тяжести инсомнии)

Тип поля: **шкала с баллами**, 7 пунктов, для каждого — выпадающий список (`select`, атрибут `data-scale="isi"`). Диапазон итогового балла: 0–28.

Полный список пунктов (одинаковая шкала ответа для каждого):

1. Трудность засыпания
2. Трудность поддержания сна
3. Ранние пробуждения
4. Удовлетворенность текущим сном
5. Заметность проблемы для окружающих
6. Беспокойство из-за нарушения сна
7. Влияние на дневное функционирование

Шкала ответа (единая для всех 7 пунктов, значения `select`):
- 0 — нет
- 1 — слабо
- 2 — умеренно
- 3 — значительно
- 4 — выражено

Итоговый балл `isi-score` (в шапке страницы) — автоматическая сумма всех 7 выборов (макс. 28), пересчитывается при любом изменении формы (`updateScores()` / `sumScale('isi')`).

Интерпретация итогового балла (`classifyIsi`, используется в тексте заключения, но не отдельное поле формы):
- ≥ 22 — тяжелая инсомния
- 15–21 — умеренная инсомния
- 8–14 — субпороговая инсомния
- < 8 — клинически значимой инсомнии по ISI не выявлено

### 3.2. ESS — Epworth Sleepiness Scale (Шкала сонливости Эпворта)

Тип поля: **шкала с баллами**, 8 пунктов, `select`, атрибут `data-scale="ess"`. Диапазон итогового балла: 0–24.

Полный список пунктов (вероятность задремать в ситуации):

1. Сидя и читая
2. Просмотр телевизора
3. Пассивно сидя в общественном месте
4. Пассажиром в машине около часа
5. Лежа днем после обеда
6. Сидя и разговаривая
7. Сидя спокойно после обеда без алкоголя
8. В машине при остановке в пробке

Шкала ответа (единая для всех 8 пунктов):
- 0 — не заснул бы
- 1 — малая вероятность
- 2 — умеренная вероятность
- 3 — высокая вероятность

Итоговый балл `ess-score` — автосумма всех 8 пунктов (макс. 24).

Интерпретация (`classifyEss`):
- ≥ 16 — выраженная дневная сонливость
- 11–15 — повышенная дневная сонливость
- < 11 — значимой дневной сонливости по ESS не выявлено

### 3.3. STOP-Bang (скрининг риска обструктивного апноэ сна)

Тип поля: **шкала с баллами**, реализована как 8 независимых чекбоксов «да/нет» (`data-scale="stopbang"`, атрибут `data-stopbang-code`). Каждый отмеченный пункт = 1 балл. Диапазон итогового балла: 0–8.

Полный список пунктов:

| Код | Формулировка | Особенность |
|---|---|---|
| S | Громкий храп | обычный чекбокс |
| T | Усталость / дневная сонливость | обычный чекбокс |
| O | Наблюдались остановки дыхания | обычный чекбокс |
| P | Артериальная гипертензия | обычный чекбокс |
| B | ИМТ > 35 кг/м² | **автоматически отмечается**, если рассчитанный ИМТ (поле `bmi`) > 35; ручное снятие галочки возможно, но при пересчете (изменение веса/роста) состояние будет переустановлено |
| A | Возраст > 50 лет | обычный чекбокс (возраст отдельным полем не собирается — вводится вручную как факт) |
| N | Окружность шеи > 40 см | **автоматически отмечается**, если поле `neck` > 40 |
| G | Мужской пол | обычный чекбокс |

Итоговый балл `stopbang-score` — автосумма отмеченных пунктов (макс. 8).

Интерпретация (`classifyStopBang`):
- ≥ 5 — высокий риск ОАС
- 3–4 — промежуточный риск ОАС
- < 3 — низкий риск ОАС

### 3.4. Индикаторы баллов в шапке страницы

`#isi-score`, `#ess-score`, `#stopbang-score` — не поля ввода, а вычисляемые индикаторы (обновляются при каждом `change`/`input` в форме через `updateScores()`), отображаются в верхней панели независимо от раздела «Шкалы».

## 7. Служебный виджет: выбор времени (sections/time_picker.html)

Не является отдельным полем данных. Модальное окно с циферблатом (часы/минуты), которое перехватывает клик/фокус на **любом** поле `input[type="time"]` формы (`bedtime`, `waketime`, а также шесть полей `data-substance-time` в блоке «Лекарства и вещества») и подставляет выбранное значение обратно в это поле в формате `ЧЧ:ММ`. Есть кнопка «Очистить» (сбрасывает значение поля) и «Готово» (подтверждает выбор). Для интеграции с МИС релевантен только конечный формат данных полей `type=time` — `ЧЧ:ММ` (24-часовой формат), сам виджет переносить не требуется.

Бодрствование после засыпания — суммарное время бодрствования после первого засыпания и до окончательного утреннего пробуждения. TST рассчитывается как время от попытки уснуть до подъёма минус латентность и это время бодрствования. При интервале более 16 часов или если сумма латентности и бодрствования после засыпания больше времени в постели, выводится предупреждение.

### Советник

`evaluateRouter(state)` возвращает только синдромные сигналы: `insomnia`, `osa`, `rls`, `hypersomnia`, `circadian`, `parasomnia`. Результат содержит статус, недостающие данные, альтернативы/коморбидность, красные флаги и идентификаторы предлагаемого плана. В диагноз данные не записываются. Чекбоксы плана изменяет только `applyRouterPlan()` после нажатия врачом кнопки «Добавить предложенный план».

### Лекарства и вещества

Строки создаются из `MEDICATION_GROUPS` и сохраняют общий контракт: `name="substance_option"`, `data-substance-id`, `data-substance-amount`, `data-substance-time`. `data-substance-time` — фактическое время последнего приёма. В интерфейсе представлены основные классы, влияющие на сон: стимуляторы и симпатомиметики; психотропные; сердечно-сосудистые, включая β-блокаторы; респираторные/противовоспалительные; неврологические и анальгетические; алкоголь и другие вещества. Поле `substances_extra` обязательно сохраняется для препаратов вне справочника.

### Печатный дневник

Если выбрана рекомендация `sleep_diary`, `generateReport()` после всех разделов добавляет приложение `sleepDiaryAppendix()` — пустую таблицу на 14 дней. Приложение начинается с новой страницы при печати и не добавляется, если рекомендация не выбрана.
